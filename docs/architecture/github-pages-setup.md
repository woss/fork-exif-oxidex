# GitHub Pages Setup Documentation

This document describes the GitHub Pages configuration for OxiDex, specifically for the tag comparison reports.

## Overview

GitHub Pages serves the OxiDex documentation and tag comparison reports at [oxidex.net](https://oxidex.net).

### Key Components

1. **Main Site** (VitePress)
   - Location: `docs/` directory
   - Built with VitePress
   - Deployed from `.vitepress/dist`
   - Workflow: `.github/workflows/deploy-docs.yml`

2. **Tag Comparison Reports** (GitHub Actions)
   - Location: `docs/reference/comparison/`
   - Generated by: `.github/workflows/compare-exiftool.yml`
   - Deployment method: `peaceiris/actions-gh-pages`
   - Deployed to: `/tag-comparison/` subdirectory

## Configuration Details

### GitHub Pages Source

**Location:** Repository Settings → Pages

**Current Configuration:**
- Source: GitHub Actions
- Build and deployment: Automated via workflows
- Domain: oxidex.net (via CNAME)

### Workflows

#### 1. Main Documentation Deployment

**File:** `.github/workflows/deploy-docs.yml`

```yaml
name: Deploy Documentation
on:
  push:
    branches: [main]
    paths:
      - 'docs/**'
      - '.github/workflows/deploy-docs.yml'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm install
      - run: npm run docs:build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs/.vitepress/dist
```

#### 2. Tag Comparison Report Deployment

**File:** `.github/workflows/compare-exiftool.yml`

```yaml
name: Generate Tag Comparison Report
on:
  push:
    branches: [main]
    paths:
      - 'src/parsers/**'
      - 'src/bin/tag-comparison/**'
  schedule:
    - cron: '0 2 * * 0'  # Weekly
  workflow_dispatch:

jobs:
  generate-report:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pages: write

    steps:
      - uses: actions/checkout@v4
      # ... build and test steps ...
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs/reference/tag-comparison
          destination_dir: tag-comparison
```

## Deployment Paths

### Main Documentation

- **Local path:** `docs/.vitepress/dist/`
- **Remote path:** Root (`/`)
- **URL:** `https://oxidex.net/`

### Tag Comparison Reports

- **Local path:** `docs/reference/comparison/`
- **Remote path:** `/tag-comparison/`
- **URL:** `https://oxidex.net/tag-comparison/`

## CNAME Configuration

**File:** `docs/public/CNAME`

```
oxidex.net
```

This file is:
- Served from the `public/` directory
- Deployed to GitHub Pages
- Used by GitHub to configure DNS

**To update domain:**
1. Edit `docs/public/CNAME`
2. Update DNS CNAME record pointing to `<username>.github.io`
3. Commit and push changes

## Permissions

### GitHub Token

The workflows use `${{ secrets.GITHUB_TOKEN }}` which has:
- Read access to repository contents
- Write access to GitHub Pages
- Permissions configured in workflow file via `permissions:` block

### Required Permissions

```yaml
permissions:
  contents: write      # Read repository contents
  pages: write         # Deploy to GitHub Pages
```

## Security Considerations

### Token Access

- `GITHUB_TOKEN` is automatically available in workflows
- Scoped to the repository
- Expires after workflow run
- No manual credential needed

### Branch Protection

- Deployments from `main` only
- Requires workflow passes
- All checks must succeed

### DNS Security

- CNAME points to `<username>.github.io`
- GitHub manages HTTPS certificates
- Automatic renewal via Let's Encrypt

## Troubleshooting

### Site Not Updating

**Check:**
1. Latest workflow run status
2. Build succeeded (`npm run docs:build`)
3. Artifacts deployed to Pages
4. Browser cache (force refresh)

**Solution:**
```bash
# Trigger manual rebuild
gh workflow run deploy-docs.yml
```

### Report Not Appearing

**Check:**
1. `compare-exiftool.yml` executed successfully
2. `destination_dir: tag-comparison` is correct
3. Files were copied to `docs/reference/comparison/`
4. HTML/JSON files generated

**Solution:**
```bash
# Manually trigger comparison workflow
gh workflow run compare-exiftool.yml
```

### Domain Issues

**Check:**
1. `docs/public/CNAME` contains correct domain
2. DNS CNAME record points to GitHub Pages
3. No trailing characters in CNAME file

**Verify DNS:**
```bash
nslookup oxidex.net
dig oxidex.net CNAME
```
