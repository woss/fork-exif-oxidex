ARCHITECTURAL CHANGES
=====================

Before:
- Ricoh: Decoders scattered (in parser), basic registry
- Parrot: Inline registry (difficult to modify), decoder in parser

After:
- Ricoh: Centralized decoders in registry, clean parser
- Parrot: Dedicated registry function, no inline code
- Both: Consistent factory pattern (registries/xyz::xyz_registry)

Benefits:
- Single source of truth for tag metadata
- Decoders defined once, reused everywhere
- Easy to extend with new tags
- Better separation of concerns
- Consistent with other manufacturers


MIGRATION PATH
==============

No migration required for users of these parsers.
The public API remains unchanged:
- RicohParser::new() - unchanged
- ParrotParser::new() - unchanged

Internal implementation details changed:
- Decoder locations: parser → registry
- Registry initialization: inline → factory function
- parse_entry() logic: simplified and better documented
